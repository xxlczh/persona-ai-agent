# 使用手册

## 概述

用户画像语义模型系统是一个基于 LLM 的智能用户画像生成工具。用户可以通过上传用户数据，让 AI 自动生成结构化的用户画像，并获得质量评估结果。

## 功能模块

### 1. 用户认证

- **注册**: 创建新账户
- **登录**: 登录已有账户
- **个人资料管理**: 修改用户名、邮箱
- **密码修改**: 修改账户密码

### 2. 项目管理

- **创建项目**: 新建项目并配置 LLM 提供商
- **项目列表**: 查看所有项目
- **项目详情**: 查看项目信息和统计
- **项目编辑**: 修改项目名称、描述
- **项目删除**: 删除项目及其关联数据
- **项目归档**: 归档不活跃的项目

### 3. 数据源管理

- **上传数据**: 上传用户数据文件
- **数据类型**: 支持 JSON、CSV、文本、PDF
- **数据预览**: 查看数据内容
- **数据编辑**: 修改数据内容
- **数据删除**: 删除数据

### 4. 画像生成

- **单画像生成**: 基于单个数据源生成画像
- **批量生成**: 基于多个数据源批量生成画像
- **模板选择**: 选择不同的 Prompt 模板
- **生成参数**: 配置温度、Token 限制等参数
- **生成进度**: 实时查看生成进度
- **生成结果**: 查看生成的画像内容

### 5. 画像管理

- **画像列表**: 查看项目下所有画像
- **画像详情**: 查看画像完整信息
- **画像编辑**: 手动编辑画像内容
- **画像删除**: 删除画像
- **画像导出**: 导出画像为 JSON 或 PDF

### 6. 质量评估

- **自动评估**: 生成画像时自动进行质量评估
- **手动评估**: 对已有画像进行重新评估
- **评估维度**: 完整性、一致性、真实性、可操作性
- **评估历史**: 查看评估历史记录
- **统计分析**: 查看项目画像质量统计

### 7. Prompt 模板

- **模板列表**: 查看所有可用模板
- **模板分类**: 按类别查看模板
- **创建模板**: 创建自定义 Prompt 模板
- **编辑模板**: 修改模板内容
- **删除模板**: 删除模板
- **模板预览**: 预览模板效果
- **模板测试**: 测试模板生成效果
- **默认模板**: 设置默认模板

## 使用流程

### 流程一：基础画像生成

1. **注册/登录**
   - 访问系统首页
   - 点击"注册"创建账户
   - 使用账户登录

2. **创建项目**
   - 点击"新建项目"
   - 输入项目名称和描述
   - 选择 LLM 提供商 (OpenAI/文心一言/智谱)
   - 点击"创建"

3. **上传数据**
   - 进入项目详情页
   - 点击"上传数据"
   - 选择数据类型 (JSON/CSV/文本/PDF)
   - 输入数据名称
   - 填写或上传数据内容
   - 点击"保存"

4. **生成画像**
   - 在数据列表中选择数据
   - 点击"生成画像"
   - 选择 Prompt 模板 (可选)
   - 配置生成参数 (可选)
   - 点击"生成"

5. **查看结果**
   - 查看生成的画像内容
   - 查看质量评估分数

### 流程二：批量生成画像

1. **准备多个数据源**
   - 上传多个用户数据文件

2. **批量生成**
   - 选择多个数据源
   - 点击"批量生成"
   - 选择 Prompt 模板
   - 点击"开始生成"

3. **查看进度**
   - 实时查看生成进度
   - 查看成功/失败数量

4. **查看结果**
   - 查看所有生成的画像
   - 批量导出画像

### 流程三：自定义 Prompt 模板

1. **创建模板**
   - 进入 Prompt 模板页面
   - 点击"新建模板"
   - 输入模板名称和分类
   - 编写 Prompt 内容
   - 保存模板

2. **测试模板**
   - 使用测试数据预览效果
   - 根据结果调整模板

3. **设为默认**
   - 测试通过后设为默认模板

## 数据格式

### JSON 数据格式

```json
{
  "users": [
    {
      "user_id": "001",
      "demographics": {
        "age": "25-35",
        "gender": "女",
        "location": "北京",
        "occupation": "产品经理"
      },
      "behaviors": {
        "app_usage": ["微信", "抖音", "小红书"],
        "shopping_habit": "周末购物",
        "content_preference": "科技、数码"
      }
    }
  ]
}
```

### CSV 数据格式

```csv
user_id,age,gender,location,occupation,app_usage,shopping_habit
001,25-35,女,北京,产品经理,"微信,抖音,小红书",周末购物
002,35-45,男,上海,工程师,"知乎,掘金,微博",工作日购物
```

### 文本数据格式

```
用户ID: 001
年龄: 25-35岁
性别: 女
职业: 产品经理
位置: 北京
常用APP: 微信、抖音、小红书
购物习惯: 周末购物
内容偏好: 科技、数码产品
```

## 画像结构

生成的画像包含以下维度:

### 1. 人口统计特征 (Demographic)

| 字段 | 说明 | 示例 |
|------|------|------|
| age | 年龄段 | 25-35岁 |
| gender | 性别 | 女性 |
| location | 地域 | 一线城市 |
| occupation | 职业 | 互联网从业者 |
| income | 收入水平 | 15-30万/年 |
| education | 教育程度 | 本科 |

### 2. 行为特征 (Behavioral)

| 字段 | 说明 | 示例 |
|------|------|------|
| online_behavior | 在线行为 | 高频使用社交媒体 |
| purchase_behavior | 购买行为 | 注重品质和口碑 |
| interaction_preference | 互动偏好 | 喜欢参与讨论 |
| content_consumption | 内容消费 | 偏好短视频和图文 |

### 3. 心理特征 (Psychological)

| 字段 | 说明 | 示例 |
|------|------|------|
| values | 价值观 | 追求品质生活 |
| attitudes | 态度 | 开放包容 |
| preferences | 偏好 | 简约设计风格 |
| motivations | 动机 | 自我提升 |

### 4. 需求特征 (Needs)

| 字段 | 说明 | 示例 |
|------|------|------|
| explicit | 显性需求 | 高效的工具 |
| implicit | 隐性需求 | 归属感和认同 |
| pain_points | 痛点 | 时间碎片化 |
| desires | 期望 | 更智能的推荐 |

### 5. 场景特征 (Scenarios)

| 字段 | 说明 | 示例 |
|------|------|------|
| usage_scenarios | 使用场景 | 工作沟通、日常娱乐 |
| trigger_scenarios | 触发场景 | 促销活动、朋友推荐 |
| decision_scenarios | 决策场景 | 购买决策、内容选择 |

## 质量评估

### 评估维度

| 维度 | 权重 | 说明 |
|------|------|------|
| 完整性 | 25% | 画像字段是否完整 |
| 一致性 | 25% | 各维度之间是否一致 |
| 真实性 | 30% | 是否符合实际用户特征 |
| 可操作性 | 20% | 是否具有指导意义 |

### 评分等级

| 等级 | 分数范围 | 说明 |
|------|----------|------|
| 优秀 | 90-100 | 画像质量很高 |
| 良好 | 80-90 | 画像质量良好 |
| 中等 | 70-80 | 画像质量一般 |
| 较差 | 70以下 | 需要改进 |

### 评估建议

系统会根据评估结果提供改进建议:

- 补充缺失的字段信息
- 修正不一致的特征描述
- 增强画像的真实性
- 提高画像的可操作性

## 常见问题

### Q1: 如何选择 LLM 提供商?

A: 根据您的需求和可用性选择:
- **OpenAI GPT**: 国际版，效果稳定
- **百度文心一言**: 国内版，响应速度快
- **智谱 GLM**: 国内版，性价比高

### Q2: 上传数据有什么限制?

A:
- 文件大小: 最大 50MB
- 文件类型: JSON、CSV、TXT、PDF
- 数据格式: 需要包含用户相关信息

### Q3: 生成失败怎么办?

A:
1. 检查数据格式是否正确
2. 检查 LLM API Key 是否有效
3. 查看错误提示信息
4. 尝试调整生成参数

### Q4: 如何提高画像质量?

A:
1. 提供更丰富完整的原始数据
2. 使用合适的 Prompt 模板
3. 调整生成参数 (如降低温度)
4. 手动编辑和优化画像

### Q5: 画像数据安全吗?

A:
- 所有数据存储在您的数据库中
- 不会用于模型训练
- 支持私有化部署
